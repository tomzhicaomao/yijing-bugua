## ADDED Requirements

### Requirement: 六线值计算本卦

系统 SHALL 根据 6 个爻值（6/7/8/9）计算出本卦编号。

- 值 7 或 9 MUST 记为阳爻（⚊）
- 值 6 或 8 MUST 记为阴爻（⚋）
- 6 个爻输入顺序 MUST 为从下到上：数组第 0 位是初爻，数组第 5 位是上爻
- 6 个爻组合 MUST 通过固定的 64 卦映射表映射到 1-64 的周易卦序编号
- 映射表 MUST 以三画卦上下组合为 key，不得依赖不透明的魔法数字算法
- 计算函数 MUST 为纯函数，相同输入永远返回相同结果

#### Scenario: 六爻全为少阳（7）

- **WHEN** 输入 `[7, 7, 7, 7, 7, 7]`
- **THEN** 本卦为乾（䷀），编号 1

#### Scenario: 六爻全为少阴（8）

- **WHEN** 输入 `[8, 8, 8, 8, 8, 8]`
- **THEN** 本卦为坤（䷁），编号 2

#### Scenario: 混合爻值

- **WHEN** 输入 `[7, 8, 9, 6, 7, 8]`
- **THEN** 本卦编号由阳/阴爻组合计算得出，结果可复现

#### Scenario: 输入顺序从下到上

- **WHEN** 输入 `[7, 7, 7, 8, 8, 8]`
- **THEN** 下卦为乾（☰），上卦为坤（☷），并按固定映射表返回对应周易卦序

#### Scenario: 映射表测试覆盖

- **WHEN** 运行规则引擎测试
- **THEN** MUST 至少包含乾、坤、屯、蒙、泰、否、既济、未济等已知卦例，防止上下卦或爻序颠倒

### Requirement: 动爻计算变卦

系统 SHALL 识别动爻（值为 6 或 9 的爻）并计算变卦。

- 老阴（6）MUST 在变卦中反转为阳爻
- 老阳（9）MUST 在变卦中反转为阴爻
- 少阴（8）和少阳（7）MUST 保持不变
- 无动爻时 MUST 返回 null 或无变卦

#### Scenario: 有动爻

- **WHEN** 输入 `[7, 8, 9, 6, 7, 8]`，第三爻为老阳(9)、第四爻为老阴(6)
- **THEN** 变卦的第三爻反转（阳→阴），第四爻反转（阴→阳），其余不变

#### Scenario: 无动爻

- **WHEN** 输入 `[7, 8, 7, 8, 7, 8]`，无老阴或老阳
- **THEN** 变卦为 null，动爻列表为空数组

### Requirement: 64 卦数据查询

系统 SHALL 支持按编号查询卦的完整数据，包括卦名、卦辞、爻辞。

- 查询函数 MUST 为纯函数
- 数据来源 MUST 为项目内置 JSON 文件（不可学习）

#### Scenario: 查询乾卦

- **WHEN** 查询编号 1 的卦
- **THEN** 返回 `{ name: "乾", trigramUpper: "☰", trigramLower: "☰", judgment: "..." }` 及其 6 条爻辞

#### Scenario: 查询不存在编号

- **WHEN** 查询编号 0 或 65
- **THEN** 抛出明确错误或返回 null

### Requirement: 爻辞检索

系统 SHALL 根据卦编号和动爻位置返回对应的爻辞。

- 当有多个动爻时 MUST 返回所有对应爻辞
- 当无动爻时 MUST 返回本卦卦辞
- Phase A 的动爻取用原则 MUST 明确：无动爻看本卦卦辞；一动爻重点看该爻辞；多动爻返回全部动爻爻辞并展示变卦作为趋势方向；六爻皆动时本卦与变卦并重

#### Scenario: 单动爻

- **WHEN** 本卦为 1（乾），动爻位置为 [3]
- **THEN** 返回乾卦九三爻辞

#### Scenario: 无动爻（静卦）

- **WHEN** 本卦为 1（乾），动爻位置为 []
- **THEN** 返回乾卦卦辞

#### Scenario: 多动爻

- **WHEN** 本卦为 1（乾），动爻位置为 [2, 5]
- **THEN** 返回乾卦九二与九五爻辞，并保留动爻顺序

#### Scenario: 六爻皆动

- **WHEN** 动爻位置为 [1, 2, 3, 4, 5, 6]
- **THEN** 返回全部六条爻辞，并标记该卦为六爻皆动以供展示层强调本卦与变卦并重
