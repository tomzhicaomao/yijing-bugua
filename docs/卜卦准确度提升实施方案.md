```

  乾☰ 兑☱ = 金
  离☲     = 火
  震☳ 巽☴ = 木
  坎☵     = 水

五行生克:
  生: 木→火→土→金→水→木
```

### Task 1.3: 月建与日辰获取

**文件**: 新建 `src/engine/calendar.ts`

**核心逻辑**:

```
月建（按月支）:
  寅月=正月(立春起)、卯月=二月、辰月=三月、巳月=四月
  午月=五月、未月=六月、申月=七月、酉月=八月
  戌月=九月、亥月=十月、子月=十一月、丑月=十二月

日辰（按日支）:
  六十甲子循环，需计算给定公历日期对应的天干地支

简化实现（免万年历查表）:
  - 固定基准: 1900-01-01 = 甲戌日
  - DayOffset = (targetDate - 1900-01-01) 得天数差
  - 日干支索引 = (DayOffset + 甲戌索引) % 60 → 查甲子表
  - 月建 = 根据节气的近似映射（±1天精度）
```

**导出接口**:

### Task 2.1: 大衍筮法实现

**文件**: 新建 `src/engine/yarrow-stalk.ts`

**传统大衍筮法规则** (据《易传·系辞上》第九章):

```
大衍之数五十，其用四十有九。
分而为二以象两，挂一以象三，
揲之以四以象四时，归奇于扐以象闰。
五岁再闰，故再扐而后挂。

一爻需要"三变" → 一卦六爻需要"十八变"

每一变的操作：
  1. 取49根蓍草（虚拟），分成两把（任意比例，用随机模拟）
  2. 从右手那把取1根，夹在左手小指间（"挂一"）
  3. 左手那把：每4根一数，取余数（1-4根）→ "揲四归奇"
  4. 右手那把：每4根一数，取余数（1-4根）
  5. 三部分之和（挂1 + 左余 + 右余）= 5 或 9
  6. 49 - 总和 = 续用策数，进入下一变

三变后剩余策数 → 爻值：
  36策 ÷ 4 = 9 → 老阳 (⚊○)
  32策 ÷ 4 = 8 → 少阴 (⚋)
  28策 ÷ 4 = 7 → 少阳 (⚊)
  24策 ÷ 4 = 6 → 老阴 (⚋×)

理论概率:
  老阴(6): 1/16 = 6.25%
  少阳(7): 5/16 = 31.25%
  少阴(8): 7/16 = 43.75%
  老阳(9): 3/16 = 18.75%
```

**实现策略**: 方案 C — 加权随机出结果 + 过程数据用于动画展示

### Task 4.3: Prompt 迭代优化系统

**文件**: 新建 `src/engine/prompt-optimizer.ts`

**思路**:

```
收集反馈 → 分析失败案例 → 找出 pattern → 调整 prompt → 版本对比

示例:
  Step 1: 分析发现 "timeWindow" 类 claim 的命中率最低 (35%)
  Step 2: 检查 prompts 中时间窗口相关的约束
  Step 3: 发现 prompt 对时间窗口要求太模糊："描述时间窗口"
  Step 4: 增强约束: "时间窗口须明确到周或月，不能只说'近期'"
  Step 5: 生成新 prompt 版本 v1.1，标记 changelog
  Step 6: A/B 对比: v1.0 vs v1.1 的 timeWindow accuracy
```

**非全自动**: 这是一个分析辅助工具，最终 prompt 调整由人工决策。
