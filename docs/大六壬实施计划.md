### A2 — 核心三个宗门（Day 2–4）

> ⚠️ **最关键的 2 天** — 涉害法是整个引擎最复杂的部分

| # | 步骤 | 产出物 | 测试用例 | 依赖 |

| A2.2 | 编写 `src/engine/liuren/jiuzongmen/biyong.ts` — ②比用法（阳干取阳支、阴干取阴支、多同阴返回 null） | 1 文件 | ≥5 | A2.1 |
| A2.3 | 编写 `src/engine/liuren/jiuzongmen/shehai.ts` — ③涉害法（**最复杂**：深度计算、深度相同取先出现者） | 1 文件 | ≥10 | A2.2 |

#### ① 贼克法 (zeke) 精确规则

```
输入：四课（四组上下神的生克关系）
规则：
  1. 遍历四课，找出所有"下贼上"（下神克上神）
  2. 若有"下贼上"：
     a. 仅一个 → 取下贼上的上神为初传
     b. 多个 → 返回多个候选（交由比用法决疑）
  3. 若无"下贼上"，找所有"上克下"：
     a. 仅一个 → 取上克下的上神为初传
     b. 多个 → 返回多个候选
  4. 既无下贼上也无上克下 → 返回 null

输出：{ 初传, 中传, 末传, 格局 } 或 null（表示无克，进入比用）
```

#### ② 比用法 (biyong) 精确规则

```
前提：贼克法返回了多个候选上神（多个下贼上或多个上克下）
规则：取与日干阴阳相同者
  日干为阳（甲丙戊庚壬）→ 取候选中的阳支（子寅辰午申戌）
  日干为阴（乙丁己辛癸）→ 取候选中的阴支（丑卯巳未酉亥）
  若仍有多个符合 → 返回 null（进入涉害法）

输出：{ 初传, 中传, 末传, 格局 } 或 null
```

#### ③ 涉害法 (shehai) 精确规则

```
前提：比用法返回了 null（多个候选且多与日干同阴阳）

涉害深度计算方法：
  对每个候选地支，计算其在地盘的涉害深度：
  从该地支在地盘的位置开始，顺时针数十二位
  每遇到一次"受克"（该位地支被候选地支五行所克）就记一次
    
五行相克：
  木克土、土克水、水克火、火克金、金克木

算法伪代码：
  function calculateShehaiDepth(candidateBranch, diPan, tianPan):
    depth = 0
    startIndex = diPan.indexOf(candidateBranch)
    从第 1 位到第 12 位：
      当前地支 = diPan[(startIndex + i) % 12]
      若 candidateBranch 五行克 当前地支五行 → depth++
    返回 depth

规则：
  1. 计算所有候选的涉害深度
  2. 取涉害最深者（最大值）对应的上神为初传
  3. 若深度相同 → 取在地盘上先出现的（四课中先出现者）

输出：{ 初传, 中传, 末传, 格局 }
```

#### ④ 遥克法 (yaoke)

```
前提：四课无克（贼克法返回 null）
规则：
  1. 遍历四课，找"日干遥克上神"（即日干克四课的上神）→ 取该上神为初传
  2. 若无，找"上神遥克日干"（即上神克日干）→ 取该上神为初传  
  3. 均无 → null

输出：{ 初传, 中传, 末传, 格局 } 或 null
```

#### ⑤ 昴星法 (maoxing)

```
前提：四课全为比和（四课均无克关系）
规则：
  阳日（日干为甲丙戊庚壬）：
    初传 = 酉（昴星）在天盘所临之神（即地盘酉位置的天盘神）
    中传 = 日支的上神（第三课的上神）
    末传 = 日干的上神（第一课的上神）
  
  阴日（日干为乙丁己辛癸）：
    初传 = 酉在地盘所乘之神（即天盘酉位置的地盘神的反向，实际取天盘酉的下神）
    中传 = 日干的上神（第一课的上神）
    末传 = 日支的上神（第三课的上神）

特殊情况：若初传与日干相同（伏吟昴星），特殊处理

输出：{ 初传, 中传, 末传, 格局 }
```

#### ⑥ 别责法 (bieze)

```
前提：四课中有三课相同（即两课完全重复）
规则：
  阳日 → 初传 = 日干合神（甲己合→取己寄宫未上神，乙庚合→庚寄宫申上神...）
  阴日 → 初传 = 日支的前三合（如日支丑→丑的前三合为巳酉丑→取巳上神）

  中传 = 末传 = 初传（三传相同）

输出：{ 初传, 中传, 末传, 格局 }
```

#### ⑦ 八专法 (bazhuan)

```
前提：干支同位（日干寄宫与日支相同）
规则：
  阳日 → 初传 = 日干上神，在地盘顺数第三位（从日干上神位置开始顺数三）
  阴日 → 初传 = 日支上神（第三课的上神），在天盘逆数三（从日支上神逆数三）

  中传 = 末传 = 初传

输出：{ 初传, 中传, 末传, 格局 }
```

#### ⑧ 伏吟法 (fuyin)

```
前提：天地盘完全相同（天盘未旋转/月将=占时）
规则：
  阳日 → 初传 = 日干上神
  阴日 → 初传 = 日支上神

  中传 = 初传的刑神（子刑卯、卯刑子、寅刑巳、巳刑申、申刑寅、
                         丑刑戌、戌刑未、未刑丑、辰午酉亥自刑）
  末传 = 中传的刑神

特殊情况：若中传自刑
  → 阳日取日支上神为中传，阴日取日干上神为中传
  → 若仍自刑，取日马为中传

输出：{ 初传, 中传, 末传, 格局 }
```

#### ⑨ 返吟法 (fanyin)

```
前提：天盘与地盘的十二地支完全相冲（每日支冲位）
规则：
  1. 找出四课中克日干的课（上神克日干）→ 取该上神为初传
  2. 若有多个 → 用比用法
  3. 若无克日干 → 取驿马（日马）为初传

  中传 = 初传本身在地盘所乘之神
  末传 = 中传本身在地盘所乘之神

输出：{ 初传, 中传, 末传, 格局 }
```

| # | 风险 | 严重度 | 概率 | 缓解措施 |
|:--:|------|:--:|:--:|------|
| 1 | 涉害法实现错误 | 🔴 高 | 中 | 与 kinliuren 交叉验证 ≥15 时间点，涉害专项 ≥10 测试用例 |

| 5 | 团队技能缺口（六壬知识） | 🟡 中 | 中 | types.ts 已有完整常量表，文档内有精确算法伪代码可逐行翻译 |

## 就绪条件

1. **Phase 0 入口**：`src/engine/liuren/types.ts` 已有完整类型系统，可直接在 `constants.ts` 中补完编译时矩阵
2. **Phase A 入口**：`jieqi.ts` + `tiandi-pan.ts` 已有 Step 1–2，可直接从 Step 3（四课 `sike.ts`）开始
3. **Phase B 入口**：`src/ai/double-call.ts` 架构可直接复用，只需新写 liuren 专用 prompt builder
4. **测试框架**：Vitest + jsdom 已就绪，只需在 `tests/` 下新增测试文件（匹配 `tests/**/*.test.ts` include pattern）
5. **交叉验证环境**：需要 `python3 -m venv .venv-liuren && pip install kinliuren`
