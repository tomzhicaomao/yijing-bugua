## 一、第一性原理拆解：大六壬到底是什么

### 1.1 剥离所有表象后的核心命题

大六壬的本质是一个**三层信息处理系统**：

```
输入层：时间（年月日时）
    ↓ 确定性计算
结构层：天地盘 + 四课 + 三传 + 天将 + 六亲 + 神煞
    ↓ 标准化映射
语义层：课体象意 + 毕法赋判断 + 天将含义 + 六亲关系
    ↓ 叙事合成
输出层：人类可读的断语
```

这个系统的根本特征是：**前三层完全确定性，最后一层需要判断力**。

### 1.2 为什么是"确定性"而非"随机性"

大六壬与六爻的根本区别不在于"哪个更准"，而在于**信息来源不同**：

| 维度 | 大六壬 | 六爻 |
|------|--------|------|
| 信息源 | 时间（天文现象） | 随机（铜钱物理运动） |
| 可重复性 | 同一时刻，任何人起出同一课 | 同一时刻，不同人摇出不同卦 |
| 认知模型 | "世界在这一刻的状态是确定的" | "世界的随机性可以被捕捉" |

大六壬的设计哲学是：**给定一个时空坐标，宇宙在这一刻的状态是唯一的、可计算的**。这不是迷信，这是一种世界观——与现代物理学中"给定初始条件，系统演化完全确定"的思想一脉相承。

### 1.3 确定性的边界

确定性不等于"绝对正确"。大六壬的确定性体现在：

- **计算过程确定**：同一输入 → 同一输出，无随机变量
- **规则体系确定**：九宗门取用规则是固定的，不因预测师而异
- **判断框架确定**：《毕法赋》100法是标准化的

但**解读的准确性**取决于：
- 预测师对规则的理解深度
- 对求占者具体情况的把握
- 经验积累和直觉判断

这就像下棋：棋规是确定的（计算层），棋理是标准化的（框架层），但棋力因人而异（解读层）。

## 二、三层架构设计

### 第一层：纯计算层（Deterministic Computation）

**职责**：给定时间参数，输出完整的课式结构。

**输入**：`{ date: Date, shiZhi?: Branch }`

**输出**：`LiurenPan`（完整课式）

**计算流程**（6步，每步完全确定性）：

```
1. 推算日干支     → [dayGan, dayZhi]
2. 推算时干支     → [shiGan, shiZhi]
3. 确定月将       → yueJiang（根据中气换将）
4. 月将加时       → 天地盘（天盘偏移量唯一确定）
5. 立四课定三传   → 四课 + 三传（九宗门规则唯一确定）
6. 排天将六亲     → 天将映射 + 六亲关系 + 神煞
```

**设计原则**：
- 纯函数：相同输入 → 相同输出，无副作用
- 无外部依赖：不调用 API，不读取数据库
- 可测试：384例回归测试（月将/四课/天乙贵人 384/384 一致）

**当前实现状态**：✅ 已完整实现（`src/engine/liuren/` 全部模块）

### 第三层：断卦方法论层（Interpretation Methodology）

**职责**：将标准化框架的输出，合成为人类可读的断语。

**这是AI介入的唯一层级**。

#### 3.1 断卦的标准流程

```
1. 确定占事类型（婚姻/事业/疾病/出行/诉讼等）
2. 选取核心类神（用神）
3. 分析课体格局（铸印/斩关/极阴等）
4. 匹配毕法赋法则
5. 分析天将象义
6. 分析六亲关系
7. 分析空亡影响
8. 分析神煞作用
9. 综合判断吉凶
10. 推算应期
```

#### 3.2 AI Prompt 设计原则

基于第一性原理，AI prompt 应该：

**输入给AI的**：
- 完整课式结构（JSON）
- 标准化框架层的分析结果（课体分类、毕法赋匹配、天将象义等）
- 用户的问题

**不让AI做的**：
- 不让AI重新计算课式（这是计算层的事）
- 不让AI凭空创造判断依据（框架层已提供）
- 不让AI做"自由联想"式的解读

**AI只做一件事**：将结构化的判断要素，合成为流畅的人类语言。

#### 3.3 邵彦和方法论的数字化

邵彦和的五大断卦原则，每一条都可以转化为结构化规则：

| 原则 | 数字化实现 | 复杂度 |
|------|-----------|--------|
| 一式多断 | 同一课式 + 不同年命 → 不同判断 | 高（需要用户输入生年） |
| 年命融入 | 本命/行年与四课三传的生克关系 | 中 |
| 课体与类象结合 | 课体格局 × 天将象义的交叉判断 | 中 |
| 空亡妙用 | 空亡的分类判断 + 场景化解读 | 低 |
| 应期推算 | 三传数理 + 空亡填实 + 驿马冲动 | 中 |

## 四、设计建议：从凌福之的抽象方法论学习

### 4.1 凌福之做了什么

凌福之的核心工作是：**将邵彦和散见于216个案例中的判断理法，抽象为100条标准化的判断法则**。

他的抽象方法论（5步）：

1. **拆解案例**：逐一分析课式结构和判断逻辑
2. **归纳格局**：将反复出现的课传结构模式归纳为典型格局
3. **赋文概括**：为每种格局写一句口诀
4. **附注释实例**：每法下附课体实例
5. **融入心得**：在前人学说基础上加入个人体会

### 4.2 数字化的凌福之

我们可以用同样的方法论，将《毕法赋》100法数字化：

**步骤1：定义数据结构**

```typescript
interface BiFaRule {
  id: number;                    // 1-100
  title: string;                 // "前后引从升迁吉"
  description: string;           // 补充说明
  condition: BiFaCondition;      // 匹配条件
  judgment: BiFaJudgment;        // 判断结果
  category: BiFaCategory;        // 所属类别
}

interface BiFaCondition {
  // 课传结构条件
  sanChuanPattern?: string;      // 三传模式（如"递生""递克"）
  keRelation?: KeRelation;       // 四课生克关系
  tianJiangPattern?: string;     // 天将模式
  liuQinPattern?: string;        // 六亲模式
  kongWang?: boolean;            // 是否涉及空亡
  // 更多条件...
}

interface BiFaJudgment {
  trend: '吉' | '凶' | '中性';   // 基本吉凶
  scene: Record<string, string>; // 不同占事的具体判断
  advice: string;                // 建议
}
```

**步骤2：将100法编码为规则**

```typescript

  {
    id: 1,
    title: '前后引从升迁吉',

    condition: {
      sanChuanPattern: '前后引从',
      // 具体匹配条件...
    },
    judgment: {
      trend: '吉',
      scene: {
        '官职': '升迁有望',
        '家宅': '搬迁吉利',
        // ...
      },
      advice: '利于进取',
    },
    category: '官禄功名',
  },
  // ... 其他99法

```

**步骤3：实现匹配引擎**

```typescript
function matchBiFaRules(pan: LiurenPan): BiFaMatch[] {
  return BI_FA_RULES
    .filter(rule => evaluateCondition(rule.condition, pan))
    .map(rule => ({
      rule,
      relevance: calculateRelevance(rule, pan),
    }))
    .sort((a, b) => b.relevance - a.relevance);
}
```

## 六、核心洞察

### 6.1 大六壬的数字化优势

大六壬是**最适合数字化的中国传统术数**，原因：

1. **完全确定性**：排盘算法可以100%自动化，不需要任何随机源
2. **规则高度结构化**：九宗门、毕法赋、天将象义都是明确的if-then规则
3. **输入简单**：只需要一个时间参数
4. **输出丰富**：天地盘、四课三传、天将、六亲、神煞、课体，多维度信息
5. **可验证**：同一输入 → 同一输出，可以做回归测试

### 6.2 邵彦和+凌福之给我们的启示

邵彦和的216个案例告诉我们：**断卦的准确性来自经验积累**。没有捷径，只有大量的实战验证。

凌福之的100条法则告诉我们：**经验可以被抽象为规则**。关键是从具体案例中提炼出通用的判断模式。

我们的数字化工作，本质上是在做凌福之做过的事：**将邵彦和的经验，转化为计算机可以执行的规则**。

### 6.3 AI的角色

AI在大六壬系统中的角色应该是**叙事者**，而不是**判断者**。

判断由框架层完成（课体分类 + 毕法赋匹配 + 天将象义 + 六亲分析），AI只负责将结构化的判断结果转化为流畅的人类语言。

这与邵彦和的方法论一致：他的断语之所以精准，不是因为他的"直觉"比别人强，而是因为他对规则的理解比别人深。AI可以做到同样的事——只要给它足够好的结构化输入。
